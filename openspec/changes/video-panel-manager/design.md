## Context

用户本地视频文件分布在多层级文件夹中，已按类型/主题分类，但缺乏统一视图进行快速浏览和播放。Windows 文件管理器无法提供视频专用的预览体验（缩略图、时长、播放进度）。

**约束条件：**
- 纯本地运行，不依赖云服务
- 需要支持大容量视频库（可能数千个文件）
- 缩略图生成需要 ffmpeg，首次扫描可能较慢
- 前端需要流畅的懒加载体验

## Goals / Non-Goals

**Goals:**
- 左侧树形导航展示文件夹层级，点击节点加载对应视频
- 右侧网格展示视频缩略图 + 元数据（文件名、时长、大小）
- 支持视频内嵌播放，记录播放进度
- 支持搜索过滤（文件名、分类、时长范围）
- 首次扫描后增量更新，避免重复扫描
- 响应式设计，适配不同屏幕尺寸

**Non-Goals:**
- 不支持视频转码或格式转换
- 不支持远程访问（仅限本机）
- 不支持用户系统或多租户
- 不支持视频编辑功能

## Decisions

### 1. 后端架构：Node.js + Express + Better-sqlite3

**选择：** 使用 Express 作为 Web 框架，better-sqlite3 作为数据库。

**理由：**
- Express 轻量、生态丰富，适合构建 RESTful API
- better-sqlite3 是同步 SQLite 实现，性能优于 async-sqlite3，代码更简洁
- 整个项目可以单一进程运行，无需复杂的进程间通信

**替代方案：**
- Fastify：性能更优，但生态不如 Express 成熟
- Koa：更现代化，但中间件生态较小

### 2. 前端架构：React + Vite + TanStack Query

**选择：** React 18 + Vite 构建 + TanStack Query 数据获取 + TailwindCSS 样式

**理由：**
- React 组件生态丰富，Ant Design/MUI 提供现成的树形和网格组件
- Vite 开发启动快，HMR 体验好
- TanStack Query 自动处理缓存、重试、分页
- TailwindCSS 快速构建响应式布局

**替代方案：**
- Vue 3：上手更简单，但团队 React 经验更足
- Next.js： SSR 不必要，增加复杂度

### 3. 缩略图策略：后端生成 + 文件缓存

**选择：** 后端使用 fluent-ffmpeg 截取视频关键帧，保存为.jpg 文件，路径存储在数据库。

**理由：**
- 后端生成可以复用系统 ffmpeg，比前端 @ffmpeg/ffmpeg 性能更好
- 文件缓存避免重复生成，扫描时检查缩略图是否存在
- 截取视频 10% 位置帧，避免黑屏片头

**替代方案：**
- 前端生成：增加前端复杂度，性能差
- 实时生成：每次请求都生成，性能极差

### 4. 文件夹树：懒加载 vs 预加载

**选择：** 首次加载所有一级目录，子节点展开时懒加载。

**理由：**
- 深层嵌套文件夹时，懒加载避免一次性返回大量数据
- 前端树组件（Ant Design Tree）支持懒加载模式
- 后端 API 设计为 `GET /api/folders/:id/children`

**替代方案：**
- 预加载全部：简单但数据量大时响应慢

### 5. 视频流：Range 请求支持

**选择：** 后端使用 `fs.createReadStream` 配合 HTTP Range 头，支持 Seek。

**理由：**
- 浏览器原生支持 Range 请求，可拖动进度条
- 无需一次性加载整个视频文件
- 大视频文件（GB 级）必须使用流式传输

### 6. 元数据存储：数据库 + 文件哈希缓存

**选择：** SQLite 存储视频元数据，使用文件路径 + 修改时间判断是否需要重新扫描。

**理由：**
- 避免每次启动都全量扫描
- 增量更新只扫描新增/修改的文件
- 数据库查询比分文件系统快

## Risks / Trade-offs

| Risk | Mitigation |
|------|------------|
| 首次扫描大量视频耗时过长 | 后台异步扫描，先返回已有数据；显示进度条 |
| ffmpeg 未安装导致缩略图失败 | 启动时检测 ffmpeg，缺失时提示用户安装；降级显示默认图标 |
| 大视频文件（>10GB）流式传输卡顿 | 使用 Range 请求；前端 buffer 配置优化 |
| 文件夹结构频繁变化导致缓存失效 | 定期后台重新扫描；提供手动刷新按钮 |
| 中文路径/文件名乱码 | Node.js Buffer 处理路径；数据库 TEXT 字段 UTF-8 |
| 缩略图占用大量磁盘空间 | 限制缩略图尺寸（320x180）；提供清理功能 |

## Migration Plan

**部署步骤：**
1. 安装 Node.js 18+
2. 安装 ffmpeg（Windows：choco install ffmpeg 或下载 exe）
3. 后端：`npm install` → `npm run dev`
4. 前端：`npm install` → `npm run dev`
5. 访问 `http://localhost:5173`

**回滚策略：**
- 纯前端应用，关闭浏览器即可
- 数据库文件位于 `data/videos.db`，删除即重置
- 配置文件 `data/config.json` 手动编辑
